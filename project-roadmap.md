### 6.2 应急预案

#### 场景1: 付费转化率<2%
```
触发条件: 上线后2周，付费率<2%
应对措施:
1. 紧急用户访谈，找出阻碍
2. A/B测试不同定价策略
3. 增加免费版限制，提升转化动力
4. 优化付费引导流程
5. 推出限时优惠活动
时间窗口: 2周内必须改善
```

#### 场景2: 7日留存率<20%
```
触发条件: 上线后1月，7日留存<20%
应对措施:
1. 数据分析找出流失点
2. 优化新手引导流程
3. 增加邮件/推送召回
4. 推出连续使用奖励
5. 建立用户成功体系
时间窗口: 1个月内必须改善
```

#### 场景3: 现金低于3个月 runway
```
触发条件: 现金<3个月支出
应对措施:
1. 立即启动融资
2. 削减非核心支出
3. 延缓非关键招聘
4. 寻求短期借款
5. 考虑预收款/年付优惠
时间窗口: 立即执行
```
